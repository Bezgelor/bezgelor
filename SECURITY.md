# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Bezgelor, please report it responsibly:

1. **Do not** open a public GitHub issue
2. Email security concerns to: **security@bezgelor.com**
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Resolution Target**: Within 30 days (depending on severity)

## Scope

This policy covers:
- Authentication and session handling
- Database access and data exposure
- Network protocol vulnerabilities
- Privilege escalation

## Out of Scope

- Game exploits (duping, speed hacks, etc.) - report via regular issues
- Denial of service through normal gameplay
- Social engineering

## Recognition

Security researchers who report valid vulnerabilities will be credited in release notes (unless they prefer anonymity).

## Supported Versions

| Version | Supported |
|---------|-----------|
| main branch | Yes |
| Tagged releases | Yes |
| Older commits | No |
